Summary dari Version Control System

Version Control System (VCS) adalah sistem yang membantu dalam mengelola perubahan pada dokumen, kode, dan file lainnya. VCS memungkinkan beberapa pengguna untuk bekerja pada proyek yang sama tanpa mengganggu pekerjaan satu sama lain. Dengan VCS, setiap perubahan yang dilakukan dapat dilacak, dan jika perlu, perubahan tersebut dapat dibatalkan atau dikembalikan ke versi sebelumnya.

Jenis-Jenis Version Control System

  • Centralized Version Control System (CVCS)

    • CVCS adalah model VCS yang menyimpan semua versi file di server pusat. Pengguna harus terhubung ke server untuk mengakses dan mengubah file.

  • Distributed Version Control System (DVCS)

    • DVCS memungkinkan setiap pengguna memiliki salinan lengkap dari repositori, termasuk riwayat versi. Pengguna dapat bekerja secara offline dan menggabungkan perubahan saat terhubung kembali.

  • Local Version Control System

    • Local VCS menyimpan semua versi file di komputer pengguna. Meskipun sederhana, model ini tidak mendukung kolaborasi dengan baik.

    Version Control System membantu menjaga integritas dan mempermudah kolaborasi antar pengguna.

Git

Git adalah sistem kontrol versi terdistribusi yang dirancang untuk kecepatan dan efisiensi. Ini memungkinkan banyak pengembang untuk bekerja pada proyek yang sama tanpa mengganggu satu sama lain. Git menyimpan riwayat perubahan secara lokal di setiap salinan repositori, sehingga memungkinkan pengembang untuk bekerja secara offline dan menggabungkan perubahan dengan mudah saat terhubung kembali.

Tiga Keadaan Git

  • Modified: File telah diubah, tapi belum disiapkan untuk commit.
  • Staged: File sudah ditandai untuk dimasukkan ke dalam commit (menggunakan git add).
  • Committed: Perubahan telah disimpan secara permanen di database Git lokal (menggunakan git commit).

Perintah Git

git init: Membuat repositori Git baru di direktori saat ini.
git clone [url]: Menggandakan repositori dari URL yang diberikan ke direktori lokal.
git add [file]: Menambahkan file tertentu ke staging area untuk commit berikutnya.
git commit -m "[pesan]": Menyimpan perubahan yang telah ditambahkan ke staging area dengan pesan deskriptif.
git status: Menampilkan status file di direktori kerja dan staging area.
git push [alias] [branch]: Mengirim perubahan yang telah di-commit ke repositori jarak jauh.
git pull [alias] [branch]: Mengambil dan menggabungkan perubahan dari repositori jarak jauh ke cabang lokal.
git branch: Menampilkan daftar cabang yang ada di repositori.
git checkout [branch]: Berpindah ke cabang yang ditentukan.
git merge [branch]: Menggabungkan cabang yang ditentukan ke cabang saat ini.
